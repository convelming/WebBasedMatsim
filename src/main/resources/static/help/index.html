Introduction

The purpose of this MATSim online tool is to build a MATSim scenario without programming. Users can just drag the blocks into the workspace, double click the block, set up according to the clicked block, link the blocks and you are ready to go.We fancy the idea of 'simple, yet efficient', so when building a trip chain, 'common sense' of understanding daily trips is enough(We assume that users already know some basics about MATSim, if not so, please check ). After the model has been run successfully, there are also some blocks to show some visual effects. Right now it is under development so everyone interested is welcome to try, and any suggestion would be greatly appreciated.

    Building scenarios

    Drag the modelling blocks and double click them to config, upload files or fill out forms, then link the blocks, click run button on the up-right corner, the model will run on the server. After that, using the visualization blocks to see the results. You can also download all the files (inputs, outputs) and run locally on your local computer. Just go to the download page by click user account icon on the up-right corner.

    Basic scenario components

    There are three basic components building a MATSim scenario: network part; plan parts and config part. The network part prepares network and traffic zone(region) data; plan parts specify trip chains of travel agents,and config part is used to config the scnario, upload files for sophisticated models.

    Network components

    MATSim requires a .xml network, and users can build the network part through: , ,

  • If you have a MATSim network already, upload it with and you are done with the network part;
  • If you have a .shp format map, you might need to use GIS tools to modify it first and then upload with , The platform will transfer it into a standard .xml network, after click the running button , you can also download it in your account.
  • If you don't have any network at hand, openstreet map is a good source of getting one. Use , to specify the map boundary and it will download automatically. Once again, after clicking the running button you can also download it in your account.

    • Beside, there is another block you might need.
    In this block users need to upload a .shp file polygons which indicates traffic zones and zone ids for later use. Because MATSim needs origins to specify their exact coordination, sometime OD matrix data only indicates to a zone level. In this case, the region block is needed to tell the platform ids of each zone and it will generate a random coord within according to the zone Id.

    Plan components

    There are actually two ways to formulate plans for MATSim, through , and trip chain blocks. If you don't have exact coord information in those, please remember to use and all region ids filled should be consistent with the ids in the . When using , please note there are traditional od matrix and trip chains to specify first. for more details, go to . Another way to create some population is to create some trip chains(try to imagine a trip using common sense, there must be a home location, some intermediate destinations and then go back home the last).By the way, users are do not have to add home again in the end. Each chain starts with , via or , and in between is , , . These three can be linked in any orders and even be skipped(there are default values for them). Again: if the travel plan is based on an od matrix, or trip chains with activity location based on traffic Zone Ids, Region block is needed, which users need to specify traffic zone id in the shp file and upload the shp file. System will generate random point in the specified Zones automatically.

    Plan components

    For advanced users, .xml plan file can always be used. For others, a batch way to input agents plans are using od matrices or od chain arrays. Upload an excel or text file according to the following headers:
    id(0)|count(1)|oriDepatrueTime(2)|origionRegion(3)|originCoordX(4)|originCoordY(5)|departureTime1(6+8*i)|desiredMode1(7+8*i)|tripPurpose1(8+8*i)|desFacilityId1(9+8*i)| facilityChangable1(10+8*i)|desRegionId1(11+8*i)|desCoordX1(12+8*i)|desCoordY1(13+8*i)
    Use to validate and edit your data.

    network block

    Accept from:

    Could link to: ,

    This block deals with network shape files. Because MATSim network links contain information of free speed, lanes, etc. These information have to be pointed out to a corresponding field in the shape file.

    It is strongly recommended editing the corresponding field of the shape file first before uploading. Note that a complete shape file has three mandatory files with the file extension of.shp, .shx and .dbf. What is more, at the moment only WGS 84(EPSG:4326) is supported and the shape file must be in polyline format.

    openstreet map block

    Accept from: none

    Could link to: ,

    Open street map is kinda coming-to-the-rescue hero for those who simply just need a map either for learning or doing serious work. Go to openstreet map, select a desired area, record its boundary and fill the blanks with the coord and it is ready to go.
    Please only select the boundary needed, excessive boundary needs longer time to download from the open street server and can also impedes dramatically later procedures.
